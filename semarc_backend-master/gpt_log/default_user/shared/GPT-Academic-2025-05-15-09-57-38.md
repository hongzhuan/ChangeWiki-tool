## 模块 libuv_Core_and_Platform-Specific_Functionalities 的文字总结

# 变更分析报告

---

### 1. 模块变更

- 更新到版本 1.48.0（稳定）。
- 合并了从 GHSA-f74f-cvh7-c6q6 的请求。
- 在停止时重置计时器队列（适用于 Unix 和 Windows）。
- 恢复 Unix 中信号处置到之前的状态。
- 修复了 Windows 的 ESRCH 实现问题。
- 可选择性地在 Windows 中无文件扩展名的情况下运行可执行路径。
- 优化了 Unix 中的 TCP 保活相关 CPP 指令。
- 对于不支持 io_uring 的 Linux 文件系统操作进行重试。
- Solaris 上支持完整的 TCP 保活。
- 将 Windows 中的 C99 注释替换为 C89 注释。

---

### 2. 功能特性变更总结

- AIX 和 IBM i 的网络接口地址功能使用 `uv_interface_addresses` 替换 `getifaddrs`。
- Unix 中忽略 NULL `ifa_addr` 的接口地址。
- 增加了对 Loongarch 架构的支持。
- 新增了 UV_PIPE_NO_TRUNCATE 标志用于 Unix 和 Windows。
- 为 Linux 增加了一些基于 io_uring 的文件系统操作。

---

### 3. 非功能特性变更总结

- 修复了 Unix 和 Windows 中的忙循环问题，尤其是零超时计时器。
- 在 MinGW 的构建过程中处理缺失的 `uuid.dll` 问题。
- 在 Windows 中尊重 NoDefaultCurrentDirectoryInExePath 环境变量的设置。
- 修复了较低版本 Android 的构建问题。
- 在 Unix 中在 fork 后重置信号计数器。
